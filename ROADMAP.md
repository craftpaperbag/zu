# ROADMAP — zu の UX 改善

このサイトを「もっと伝わる・もっと持っていきやすい」ものにするための改善リスト。
思想（`CLAUDE.md`）と照らして、効果の大きい順に並べてある。進捗はチェックボックスと状態で管理する。

## 状態の見かた

- `[ ]` 未着手 / `[x]` 完了
- **状態**：`未着手` → `着手中` → `完了`（必要なら `見送り`）
- 利用者に見える変更を入れたら、`history.js` への追記も忘れない。

---

## 優先度：高

- [x] **1. 個別Tipへの共有リンク（ディープリンク）** — 状態：完了（2026-06-20）
  モーダルの開閉・検索・絞り込みが URL に残らず、「この一枚いいね」を人に渡せない。
  核は「全部開示して、使えそうなら持っていってください」なのに、今はサイト全体しか持っていけない。
  → `#tip=<id>` で詳細を復元、`?cat=&q=` で絞り込みも共有可能にした。共有リンク直開きでも閉じてサイト内に留まる。

- [x] **2. 一覧カードに `why`（このサイトの命）を出す** — 状態：完了（2026-06-20）
  `why` と `visual` が命、と `CLAUDE.md` 自身が言う。なのに一覧では `why` が隠れていて、
  スクロールするだけの大半の訪問者がサイト最大の魅力に触れない。
  → カードに `why` の一文を2行（`.clamp2`）で添えた。

- [x] **3. モーダル内の前後送り（← →）** — 状態：完了（2026-06-20）
  63種あるのに、詳細を見るたび閉じて開き直す必要がある。
  → モーダル下部に前/次ボタンと位置表示、矢印キー対応を足した。絞り込み中の並びをたどる。

- [x] **4. 朱（accent）を「ここぞの一点」に戻す** — 状態：完了（2026-06-20）
  `claim` を全カードで朱字表示していて、図の中の "ここぞの一点" が埋もれる。
  サイト自身が「朱は一点だけ」の原則を破っている。
  → カード・詳細の `claim` を `--ink` にし、朱は図の中の焦点に一本化した（`CLAUDE.md` のスキーマ注記も更新）。

## 優先度：中

- [x] **5. アクセシビリティの穴** — 状態：完了（2026-06-20）
  - モーダルにフォーカストラップがない（Tabで背後へ抜ける）。
  - 「用語：」の `#9aa0ab` は白地でコントラスト不足（WCAG AA未達）。
  → 詳細・履歴の両モーダルで Tab を中だけに巡らせるトラップを足し、「用語：」を `--ink-soft` に直して生hexも解消した。

- [ ] **6. タグを本当の横串にする** — 状態：未着手
  `tags` の中身がカテゴリIDそのもので、横串の意味が薄い。
  カテゴリを跨ぐ細かいテーマタグ（例：before-after、余白）を設計すると回遊が増える。

## 進め方

- 制約は常に「ビルドなし・バンドラ不要」を守ること。本体は `index.html`、データは `tips.js`（種）と `history.js`（更新履歴）に分け、`<script src>` で読み込む構成。この分け方は崩さない。
- 1項目ずつ、`history.js` への追記とセットで進める。
- 完了したらこのファイルの該当行を `[x]` ＋ 状態：完了 に更新する。
