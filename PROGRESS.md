# 仕掛かり中の作業

## 覚え書き同士の「つながり」（links）— リンク網の作成

**いま何をしているか**：67種の覚え書きを互いに結ぶリンク網の本格展開。仕組みは完成済み。全種を見渡して全体設計でリンクを張り終えた段階。**作者レビュー待ち。**

**仕組み（完成・実装済み）**
- スキーマに `links:[{to,rel}]` を追加。片側だけ書けば逆向きは自動で出る（両端に書かない）。
- `rel` は4種：`echo`（別の顔で・対称）/ `akin`（近い話・対称）/ `tension`（逆から見ると・対称）/ `then`（この次に／その前に・向きあり）。
- 定義・描画：`index.html` の `REL` / `REL_ORDER` / `BACKLINKS` / `tiesFor()` / `tiesBlock()`。詳細モーダル最下部の「つながり」節に出る。

**どこまで終わったか**：全67種にリンク網を張り終えた（2026-06-22）。手始めの7エッジ（作者承認済み）に加え、全体設計で展開。
- **検証済み（`node`＋`vm` で tips.js を読んで集計）**：壊れたリンク 0／孤立種（0本）0／degree 1 の種 0／上限(4本)超過 0。
- 1種あたりのつながり本数（前向き＋逆向きの合算・重複除去）：2本=44種、3本=22種、4本=1種（`donki` のみ）。すべて「2〜4本」の目安内。
- 種別の内訳：echo 14／akin 59／tension 3／then 3。akin はカテゴリ横断の「関連が深い」全般の受け皿なので多くなるのが自然。tension・then は性質上どうしても少ない（PROGRESS の注意どおり）。

**確定した約束（レビューで合意済み）**
- 言葉：「別の顔で／近い話／逆から見ると／この次に・その前に」で確定。
- 本数：1種あたり 2〜4本（増やしすぎない＝「増やすほど薄まる」をリンク自身でも守る）。

**次にやること**
- 作者レビュー：全体のつながりが妥当か（特に echo の当てはめと、`donki` が4本になっている点）を確認してもらう。
- スクショで実描画を確認（CDN遮断のため `SCREENSHOTS.md` の手順）。
- `tane.html`（種の工房）の生成プロンプトに `links` を含めるか要検討。

**注意点**
- リンクは片側だけ。逆向きを手で足すと二重に出る。
- 検証：`node` で `vm` に tips.js を読ませて degree（前向き＋逆向き・`to|rel` で重複除去）を集計すると、壊れ・孤立・本数超過を確認できる。
