# 仕掛かり中の作業

## 覚え書き同士の「つながり」（links）— リンク網の作成

**いま何をしているか**：67種の覚え書きを互いに結ぶリンク網を張る作業。仕組みは完成。これから全種を見渡して、つながりを本格的に張っていく。

**仕組み（完成・実装済み）**
- スキーマに `links:[{to,rel}]` を追加。片側だけ書けば逆向きは自動で出る（両端に書かない）。
- `rel` は4種：`echo`（別の側面で同じこと・対称）/ `akin`（関連が深い・対称）/ `tension`（逆から見た対の話・対称）/ `then`（前後で効く・向きあり）。
- 色は足さず、アイコン＋静かな言葉で区別（`semantic-color` を濁さない方針）。
- 定義・描画：`index.html` の `REL` / `REL_ORDER` / `BACKLINKS` / `tiesFor()` / `tiesBlock()`。詳細モーダル最下部の「つながり」節に出る。クリックで `openTip` 遷移（戻るで元へ）。
- 文書：CLAUDE.md のスキーマ＋「つながり」節に記載済み。

**どこまで終わったか**：手始めの実リンク7エッジを投入し、**作者レビュー済み・承認**（2026-06-22）。スクショで4種すべて実描画確認済み。
- `dilute` →echo→ `fewer-colors` / `few-parallels` / `one-color`
- `one-color` →akin→ `fewer-colors`
- `soft-contrast` ↔tension↔ `ensure-contrast`
- `proximity` ↔akin↔ `belonging`
- `rough-first` →then→ `show-early`

**確定した約束（レビューで合意）**
- 言葉：「別の顔で／近い話／逆から見ると／この次に・その前に」で確定。
- 本数：1種あたり 2〜4本を目安（増やしすぎない＝「増やすほど薄まる」をリンク自身でも守る）。
- `dilute` の3本 echo はこのまま採用（密だが許容）。

**次にやること（本格展開：全体設計）**
- 全67種を見渡し、リンク網を設計して張る。種別の偏り（tension・then が少なくなりがち）に注意。
- 設計の進め方：まず全エッジ案を一覧（from→rel→to＋一言根拠）で作り、張ってから検証。
- 検証：`node`＋`vm` で tips.js を読み、壊れたリンク0・孤立種（リンク0本）の洗い出し・1種あたり本数の上限超過チェックを回す。
- `tane.html`（種の工房）の生成プロンプトに `links` を含めるか要検討。

**注意点**
- リンクは片側だけ。逆向きを手で足すと二重に出る。
- 検証：`node` で `vm` に tips.js を読ませて `tiesFor` 相当を回すと、壊れたリンク・向きを確認できる。
